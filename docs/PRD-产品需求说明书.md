## 一、产品概述

### 1.1 产品定位

**义务教育优质均衡督导评估系统**是一套面向教育管理部门的综合督导评估管理平台，专门用于：
- **义务教育优质均衡** 状态评估
- **学前教育普及普惠** 督导评估

### 1.2 核心价值

| 价值点 | 说明 |
|--------|------|
| 标准化评估 | 基于国家标准的指标体系，确保评估科学规范 |
| 全流程管理 | 从数据采集到报告生成的完整业务闭环 |
| 多角色协作 | 管理员、采集员、专家、决策者分工明确 |
| 数据驱动决策 | 多维度统计分析，支持科学决策 |

### 1.3 产品范围

### 3.2 模块详细说明

#### 3.2.1 基础库管理

**指标体系库**

指标体系是评估项目的核心框架，定义了评估的维度、指标和达标标准。

*页面功能：*
| 功能 | 说明 |
|------|------|
| 指标体系列表 | 展示所有指标体系，支持搜索和状态筛选 |
| 统计卡片 | 指标库总数、已发布数、编辑中数、达标类数、评分类数 |
| 新建指标体系 | 创建新的指标体系（名称、类型、目标、描述、标签、附件） |
| 查看详情 | 查看指标体系基础信息和指标预览 |
| 编辑指标树 | 进入树形编辑页面构建指标结构 |
| 发布/取消发布 | 控制指标体系的可用状态 |
| 删除 | 删除未发布的指标体系 |

*指标体系类型：*
| 类型 | 说明 | 评估方式 |
|------|------|----------|
| 达标类 | 以达标/不达标为评估结果 | 每项指标设置达标阈值，统计达标率 |
| 评分类 | 以分数为评估结果 | 每项指标设置权重，加权计算总分 |

*指标树结构（三级）：*
| 层级 | 名称 | 说明 | 特点 |
|------|------|------|------|
| 1 | 一级指标 | 评估大类/维度 | 包含多个二级指标 |
| 2 | 二级指标 | 评估子类/方面 | 包含多个三级指标 |
| 3 | 三级指标 | 具体评估点 | 叶子节点，可配置数据指标和佐证材料 |

*指标属性：*
| 属性 | 说明 |
|------|------|
| 编码 (code) | 指标唯一标识，自动生成（如：A、A1、A1.1） |
| 名称 (name) | 指标显示名称 |
| 层级 (level) | 指标层级（1-3） |
| 权重 (weight) | 指标权重（评分类体系使用） |
| 是否叶子节点 | 标识是否为最底层指标 |
| 数据指标 | 关联的量化数据指标列表 |
| 佐证材料 | 需要提交的证明材料列表 |

*数据指标属性：*
| 属性 | 说明 |
|------|------|
| 编码 (code) | 数据指标唯一标识 |
| 名称 (name) | 数据指标名称 |
| 阈值 (threshold) | 达标阈值（如：≥90%、1:15） |
| 阈值类型 | single（单值）、range（区间） |
| 精度 (precision) | 数值精度（小数位数） |
| 目标类型 | 指标目标类型标识 |

*佐证材料属性：*
| 属性 | 说明 |
|------|------|
| 编码 (code) | 材料唯一标识 |
| 名称 (name) | 材料名称 |
| 文件类型 | 允许上传的文件类型 |
| 是否必填 | 是否为必须提交的材料 |

*指标体系状态：*
| 状态 | 说明 | 可执行操作 |
|------|------|------------|
| draft | 草稿 | 编辑、删除 |
| editing | 编辑中 | 编辑、发布 |
| published | 已发布 | 取消发布 |

*核心操作流程：*
```
创建指标体系 → 构建指标树结构 → 配置三级指标详情 → 发布指标体系
      ↓              ↓                    ↓
  选择体系类型   支持拖拽排序       配置数据指标和佐证材料
  设置基础信息   支持JSON导入       设置权重（评分类）
```

*指标树编辑功能：*
| 功能 | 说明 |
|------|------|
| 添加指标 | 在当前层级添加新指标，自动生成编码 |
| 删除指标 | 删除指标及其所有子指标 |
| 拖拽排序 | 拖动指标调整顺序 |
| 层级约束 | 最多三级，超出层级禁止添加子节点 |
| JSON导入 | 批量导入指标树结构 |
| 规范化 | 自动修正指标编码格式 |

*与其他模块关系：*
- 指标体系 ←→ 项目配置：项目关联指标体系作为评估标准
- 数据指标 ←→ 要素：数据指标引用要素获取实际数据
- 佐证材料 ←→ 数据填报：学校上传佐证文件

**要素库**

要素是评估数据的最小采集单元，通过要素库进行统一管理。

*页面功能：*
| 功能 | 说明 |
|------|------|
| 要素库列表 | 展示所有要素库，支持搜索过滤 |
| 统计卡片 | 要素库总数、已发布数、未发布数、要素总数 |
| 新建要素库 | 创建新的要素库（名称、描述） |
| 查看详情 | 查看要素库基础信息和要素预览 |
| 编辑要素 | 进入要素编辑页面管理要素 |
| 发布/取消发布 | 控制要素库的可用状态 |
| 删除 | 删除未发布的要素库 |

*要素类型：*
| 类型 | 说明 | 数据来源 |
|------|------|----------|
| 基础要素 | 直接采集的数据项 | 关联采集工具的表单字段 |
| 派生要素 | 通过公式计算的数据项 | 基于其他要素计算（如：师生比 = 教师数 / 学生数） |

*要素属性：*
| 属性 | 说明 |
|------|------|
| 编码 (code) | 要素唯一标识 |
| 名称 (name) | 要素显示名称 |
| 数据类型 | 文本、数字、日期、时间、逻辑、数组、文件 |
| 采集级别 | school（学校级）、district（区县级）、auto（自动判断） |
| 关联工具 | 基础要素关联的采集工具ID |
| 关联字段 | 基础要素关联的表单字段ID |
| 计算公式 | 派生要素的计算公式 |
| 聚合配置 | 多填报数据的汇总方式（求和、平均、计数、最大、最小、中位数、标准差、极差、差异系数） |

*核心操作流程：*
```
创建要素库 → 添加要素 → 配置要素属性 → 关联采集工具字段 → 发布要素库
                ↓
           派生要素：配置计算公式
                ↓
           多填报要素：配置聚合方式
```

*与其他模块关系：*
- 要素 ←→ 采集工具字段映射：基础要素关联表单字段，实现数据采集
- 要素 ←→ 数据指标关联：要素为数据指标提供数据支撑
- 要素 ←→ 派生计算：派生要素引用其他要素进行计算

**采集工具库**
- 功能：设计数据采集表单
- 核心操作：创建工具 → 设计字段 → 配置逻辑 → 发布
- 工具类型：表单、问卷

**用户账号管理**
- 功能：管理系统用户账号
- 分类：学校账号管理、专家账号管理

#### 3.2.2 项目管理执行

**项目创建与配置**

项目配置中心包含 5 个核心 Tab：

| Tab | 名称 | 功能 |
|-----|------|------|
| 1 | 评估对象 | 配置参评区县和学校 |
| 2 | 填报账号 | 配置项目人员（管理员、采集员） |
| 3 | 指标体系 | 查看和调整项目指标权重 |
| 4 | 填报任务 | 为采集员分配填报任务 |
| 5 | 专家评估 | 为专家分配评估任务 |

**项目状态流转**

```
配置中 ──发布──→ 填报中 ──进入评审──→ 评审中 ──完成──→ 已完成
   ↑                │                  │
   └────── 重启 ────┴───── 中止 ──────→ 已中止
```

#### 3.2.3 数据采集填报

**采集员工作台**
- 查看分配的项目和任务
- 任务状态：待开始 → 进行中 → 已完成/已逾期

**动态表单填报**
- 根据采集工具定义动态渲染表单
- 支持草稿保存
- 支持附件上传

**问题整改**
- 查看被驳回的数据
- 根据审核意见修改
- 重新提交

#### 3.2.4 专业评估评分

**专家评估工作台（5 个子 Tab）**

| Tab | 功能 | 核心内容 |
|-----|------|----------|
| 指标评分 | 逐项评分 | 指标树、评分面板、评分依据 |
| 数据查阅 | 查看数据 | 学校信息、指标数据、填报记录 |
| 佐证材料 | 材料查看 | 分指标展示、预览下载 |
| 评价意见 | 撰写评价 | 优势、不足、建议、亮点 |
| 评估结论 | 生成结论 | 综合得分、等级、达标判定、问题台账 |

**问题台账**
- 登记发现的问题
- 设置整改期限
- 跟踪整改状态
- 复评验收

#### 3.2.5 数据审核验收

**数据审核界面**
- 统计卡：待审核数、已批准数、已驳回数
- 提交列表：项目、学校、工具、状态
- 操作：查看、批准、驳回

**驳回机制**
- 填写驳回原因
- 设置整改期限
- 通知采集员

#### 3.2.6 报告与决策支持

| 报告类型 | 功能 | 数据来源 |
|----------|------|----------|
| 统计报告 | 全局数据统计 | school_indicator_data, district_statistics |
| 排名报告 | 学校/区县排名 | evaluation_results（综合得分） |
| 预警报告 | 未达标预警 | is_compliant = false 的指标 |
| 对比报告 | 对象对比分析 | 学校/区县数据对比 |
| 项目报告 | 完整评估报告 | 汇总所有数据（可导出 PDF） |

### 9.3 数据库初始化

```bash
# 1. 创建核心表结构
psql -f backend/database/schema.sql

# 2. Supabase 初始化
psql -f backend/database/supabase-setup.sql

# 3. 执行迁移脚本
psql -f backend/database/migrations/*.sql

# 4. 功能扩展（如学前教育支持）
psql -f backend/database/add-preschool-support.sql
```
